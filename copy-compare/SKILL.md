# 落地页文案竞品对比分析（直接回应文案专家版）

## 概述

从"谁的文字更能卖货"的角度，对**版本A（我方文案）**与**版本B（竞品文案）**进行深度对比和量化打分（满分 100），输出对比报告和针对版本A的具体优化建议。

## 使用方式

```
/copy-compare
/copy-compare output/optimized.md input/competitor-copy.md
/copy-compare output/final.md https://competitor-url.com
```

## 输入来源

- **版本A（我方）**：默认 `output/optimized.md`（Landing Page 终审通过稿）；或用户指定的 `.md`/`.html`
- **版本B（竞品）**：默认 `input/competitor-copy.md`（用户手动粘贴）；或 URL（WebFetch 抓取失败时用 Chrome/Playwright 兜底）

---

## 角色设定

你现在是一位顶级的"直接回应文案专家（Direct Response Copywriter）"。你只评判文案本身的销售杀伤力——即文字、结构、叙事对读者心理的操控能力。价格策略、套餐组合、退款政策等商业运营决策不在评判范围内。

## 业务背景

纯卖货的护肤及功能性产品电商网站（品类：丰唇、脱毛、紧致、瘦身、疼痛缓解等）。

## 目标受众画像

年龄段：**45 岁-50 岁+** 的中老年男性女性为主。核心特征：

- **极度务实**：只关心"这东西对我有什么用"、"能不能解决我的痛苦"
- **防备心极重**：怕被骗，怕没效果，极度依赖真实评价和权威背书
- **注意力稀缺**：没耐心看长篇大论，喜欢大白话、直接给结论
- **决策靠情绪+信任**：不是被逻辑说服，是被画面感和"跟我一样的人"打动

---

## 重要排除项（不纳入评分）

- 定价策略（单价、套餐阶梯、折扣幅度、价格锚定）
- 退款/保证政策
- 支付方式与物流承诺
- 页面框架结构（Section 数量、顺序、功能定位属于运营决策，不因框架差异本身扣分或加分）

## 打分校准

- **70 分** = 行业平均水平的电商落地页文案
- **80 分** = 明显高于平均、有多处可圈可点的亮点
- **90 分+** = 极少出现，几乎每个维度都接近最优解

## 引用规则

每个维度的对比分析中，必须各引用版本A和版本B的**至少 1 句原文**作为论据。不允许脱离原文泛泛而谈。

---

## 纯文案销售力打分维度（总分 100）

### 维度1：黄金开头与注意力锁定（20分）
- 标题和前 3 屏内容是否瞬间抓住 50 岁+人群的眼球？
- 是否一针见血指出了目标人群最痛的痛点？
- 标题读完后，读者是否有强烈冲动继续往下滑？

### 维度2：信任链构建（25分）
- 权威背书是否可信且与目标人群认知匹配？
- 用户证言是否具备"粗糙的真实感"（年龄/职业/城市颗粒度、像真人说话）？
- 整体信任体系能否瓦解 50 岁+"怕被骗"心理？

### 维度3：利益翻译与欲望激发（30分）——【核心文案分，权重最高】
- 产品卖点是否翻译成目标人群秒懂的"大白话利益"？
- 是否创造了具体的"未来画面"让读者代入？
- 成分/机制的科学信息是否做了"降维翻译"？
- 读完后，读者是否产生"我想变成文案里描述的那个人"的欲望？

### 维度4：叙事结构与阅读推进力（15分）
- 是否有清晰情绪曲线（痛点→共鸣→希望→证据→想象→行动）？
- 段落之间是否有"钩子"让读者持续往下滑？
- 信息密度是否适配目标人群？
- （注：两个版本可能采用不同页面框架结构，此维度只评判各自框架下的文案执行质量，不因框架差异本身扣分或加分。）

### 维度5：行动号召与临门一脚（10分）
- CTA 文案是否清晰、有行动感、有情绪驱动力？
- 下单路径在文案层面是否显得极其简单？
- 文案结尾是否给了读者"现在就行动"的情绪推力？

## 输出格式（严格按此结构）

### 第一部分：打分表

| 维度 | 版本A（我方） | 版本B（竞品） | 差距 |
|------|-------------|-------------|------|
| 1. 黄金开头与注意力锁定（20分） | X | X | +/-X |
| 2. 信任链构建（25分） | X | X | +/-X |
| 3. 利益翻译与欲望激发（30分） | X | X | +/-X |
| 4. 叙事结构与阅读推进力（15分） | X | X | +/-X |
| 5. 行动号召与临门一脚（10分） | X | X | +/-X |
| **总分（100分）** | **X** | **X** | **+/-X** |

### 第二部分：关键 PK 对比

逐维度直接对比，每个维度用 2-3 句话说清楚谁赢、为什么赢。打分严苛，不允许礼貌性给高分。每个维度必须各引用版本A和版本B的至少 1 句原文作为论据。

### 第三部分：最终判定

明确指出从纯文案销售力角度，版本 X 更能卖货。用一句话总结其致命优势。

### 第四部分：我方版本优化建议

⚠️ **硬性约束**：版本A的落地页框架（Section 数量、顺序、功能定位）是固定的，**不可**建议增删、合并、拆分或重新排列 Section。所有优化建议必须在现有框架内部进行，只优化具体的文案措辞、句式、修辞手法、情绪节奏。

- **补短板**：版本A 在哪个维度的【文案表达】失分最多？在不改变该 Section 功能定位的前提下，给出一条具体改写方案（精确到哪个 Section、原文是什么、改成什么、为什么能提分）。
- **抄优点**：版本B 的文案中有哪一个具体的【遣词造句技巧或修辞手法】值得版本A 借鉴？（句子级别的写法：比喻、对比、口语化等；不包括叙事策略或开头方式的改变。）给出在版本A 现有框架内的具体改写示例。

**不要**给版本B任何优化建议。

---

## 输出文件

保存完整对比报告到 `output/compare-result.md`。

## Step 5：询问用户

报告输出后询问：
- "需要我根据对比结果直接优化我们的文案吗？" → 若是，建议走 `/copy-optimize`
- "需要针对某个具体维度深入分析吗？"

## 抓取容错（竞品 URL 场景）

按优先级依次尝试：
1. WebFetch 直接抓取 URL
2. 返回 403/超时/失败 → 使用 `mcp__Claude_in_Chrome` 或 Playwright 浏览器工具
3. 浏览器工具也不可用 → 提示用户手动粘贴到 `input/competitor-copy.md`

每个 URL 必须确认拿到实际文案内容后再进入对比环节，不能跳过。
